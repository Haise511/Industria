# Индустрия — Telegram Mini App

Мини-приложение для платформы «Индустрия». Стек: **React 18 + TypeScript + Vite**.

## Возможности

В приложении реализованы следующие экраны (всего ~14 уникальных макетов из 46 фреймов Figma; повторяющиеся состояния скомпонованы в общие компоненты):

- **Онбординг**: выбор языка → выбор роли → заполнение профиля → загрузка
- **Лента** (`/feed`): поиск с переключателем «Обычный / Тойский», чипы фильтров, карточки заказов, плавающая кнопка «Создать заявку», нижняя навигация
- **Детали заказа** (`/feed/:id`): крупный аватар автора, описание, таблица параметров, кнопка «Откликнуться»
- **Дата** (`/respond/date`, `/create/date`): календарь на месяц с выбором даты + комментарий
- **Подтверждение** (`/respond/confirm`, `/create/confirm`): сводка перед отправкой
- **Создать заявку** (`/create`): форма с описанием, ценой, городом, режимом
- **Заявки** (`/orders/my`): список созданных заявок
- **Отклики** (`/responses`): отклики со статусами «В ожидании / Принят / Отклонен»
- **Заказы** (`/active`): активные заказы с действиями принять/отклонить
- **Профиль** (`/profile`): карточка с аватаром, прогресс заполненности, меню
- **Уведомления** (`/notifications`): сгруппированные по дням
- **Фильтр** (`/filter`): роль, диапазон цены, рейтинг, договор, верификация
- **История заказов** (`/history`)
- **Верификация** (`/verification`): описание процесса + CTA

## Дизайн-токены

Все цвета, шрифты и радиусы взяты 1:1 из Figma. Источник истины — `src/styles/tokens.css`. Брендовые цвета **зафиксированы** и не зависят от темы Telegram (`themeParams`), как и предусмотрено макетом.

| Токен | Значение |
|---|---|
| `--bg` | `#1d1d1d` |
| `--surface-1` (карточка) | `#272727` |
| `--surface-2` (профиль) | `#242424` |
| `--surface-pill` (топ-бар) | `#413f40` |
| `--brand` (CTA) | `#4f62ec` |
| `--brand-soft` (прогресс) | `#7584f3` |
| `--success` | `#34c759` |
| `--danger` | `#ff5356` |
| `--star` | `#fbbe25` |

**Шрифт:** в Figma используется коммерческий **Fixel Display** (https://fixelfont.com). Для разработки подставлен open-source аналог **Manrope** через Google Fonts. После приобретения лицензии замените ссылку в `index.html` на самохостинг Fixel Display, токены менять не требуется.

## Запуск локально

```bash
cd industria
npm install
npm run dev
```

После запуска откройте http://localhost:5173. В обычном браузере приложение работает как обычный SPA. Для тестирования внутри Telegram нужен HTTPS-туннель (см. ниже).

### Проверка в Telegram локально

Для тестирования внутри клиента Telegram приложение должно быть доступно по HTTPS. Используйте `cloudflared` или `ngrok`:

```bash
# вариант 1: cloudflared (без аккаунта)
cloudflared tunnel --url http://localhost:5173

# вариант 2: ngrok
ngrok http 5173
```

Полученный HTTPS-URL подставьте в настройки Mini App у бота (см. секцию ниже).

## Создание Telegram-бота и регистрация Mini App

1. Откройте чат с **@BotFather** в Telegram.
2. `/newbot` → задайте имя и username для бота (например, `industria_app_bot`).
3. Сохраните выданный **HTTP API token** (понадобится только если будете подключать backend).
4. `/newapp` → выберите вашего бота → заполните:
   - **Title**: «Индустрия»
   - **Short description**: краткое описание для маркетплейса Mini Apps
   - **Photo**: 640×360 png (можно временно использовать макет из `industria-build/screens/1_5957.png`)
   - **Demo GIF**: опционально
   - **Web App URL**: ваш HTTPS-URL (cloudflared/ngrok для разработки или прод-домен)
   - **Short name**: например, `industria`
5. Mini App будет доступен по ссылке `https://t.me/<bot_username>/<short_name>`.
6. Чтобы добавить кнопку «Открыть приложение» в меню бота: `/mybots` → выберите бота → **Bot Settings** → **Menu Button** → **Configure menu button** → введите URL и текст кнопки.

## Деплой в продакшн

Так как приложение — статическая SPA без backend (на этом этапе), его можно бесплатно захостить на любой из этих платформ:

### Vercel
```bash
npm install -g vercel
vercel --prod
```
В качестве **build command** укажите `npm run build`, **output directory** — `dist`.

### Netlify
1. Залейте репозиторий в GitHub.
2. На netlify.com → New site from Git → выберите репозиторий.
3. Build command: `npm run build`, Publish directory: `dist`.

### Cloudflare Pages
1. Pages → Create a project → Connect to Git.
2. Build command: `npm run build`, output: `dist`.

После деплоя обновите Web App URL в настройках Mini App у BotFather.

## Telegram Web App API

Интеграция с SDK сделана в `src/telegram.ts`:

- `WebApp.ready()` + `WebApp.expand()` — вызываются на маунте `App`.
- `setHeaderColor` / `setBackgroundColor` — синхронизируют чром Telegram с фиксированной темой `#1d1d1d`.
- `BackButton` — автоматически показывается на не-табных экранах и привязан к `navigate(-1)`.
- `HapticFeedback.impactOccurred('light' | 'medium')` — на тапах кнопок.

Главная CTA-кнопка по дизайну расположена внутри контента (плавающая или в футере экрана), поэтому **MainButton не используется** — это обоснованное решение по макету. Если в будущем какой-то из CTA нужно превратить в нативную MainButton, см. пример: `WebApp.MainButton.setText('Откликнуться'); WebApp.MainButton.show(); WebApp.MainButton.onClick(handler)`.

## Безопасность

Если приложение начнет работать с реальными данными:
- `initData` (raw query string) валидируйте на сервере с использованием HMAC-SHA256 от вашего bot token. Никогда не доверяйте `initDataUnsafe` для security-критичных операций.
- Не храните токены/секреты в клиенте.
